# 1. Introduction（序論）

## 1.1 研究の背景と動機

本研究は、以下の中心的な主張を理論的・工学的に再構成し、後続の要件定義・アーキテクチャ設計・実装仕様・API仕様の理論的根拠を提供するものである。

> **専門用語とは、複数の概念・前提・制約を、論理構造を保ったまま短い記号列へ圧縮した表現である。**

この視点は、自然言語処理における専門用語の扱いに新たな理論的枠組みを提供し、特に大規模言語モデル（LLM）における知識表現と推論の本質的理解に寄与する。

## 1.2 中心仮説

### 1.2.1 基本仮説

研究仮説は次のように表現できる。

> **専門用語 = 概念の論理的圧縮**

ここでいう「論理的圧縮」とは、単なる文字列短縮ではない。
ある高位語 \( t \) が、その背後にある概念集合 \( C = \{c_1, c_2, \dots, c_n\} \)、前提 \( P \)、推論規則 \( R \)、制約 \( \Gamma \) を保持したまま記述長を短縮する作用を指す。

### 1.2.2 統計的圧縮と論理的圧縮の統合

従来の言語モデルは統計的圧縮に特化しており、論理的制約の検証が不十分である。一方、形式検証システムは論理的厳密性を持つが、統計的な柔軟性に欠ける。本理論は、この2つのアプローチを統合する。

\[
\text{圧縮} = \text{統計的圧縮} \times \text{論理的制約}
\]

具体的には、確率的型システムを導入し、型制約を確率的に扱うことで、統計的学習と論理的検証を統合する。これにより、ハード制約（論理的必然性）とソフト制約（統計的傾向）を区別し、学習過程で動的に調整することが可能となる。詳細は第5章で述べる。

### 1.2.3 本理論における知能の操作的定義

本理論では、専門用語処理に関連する知能の側面を以下の3つの能力として操作的に定義する：

1. **抽象化能力**
   低位概念列から、適切な高位概念表現へ圧縮する能力
2. **展開能力**
   高位概念表現から、必要な低位概念・暗黙前提・含意を復元する能力
3. **制約検証能力**
   圧縮・展開のいずれにおいても、論理破壊や型不整合を検出する能力

**注**: これは知能の全体的定義ではなく、本研究の範囲内での操作的定義である。この3要素は第4章の問題定式化の直接的根拠となる。

## 1.3 研究の貢献

本研究の主な貢献は以下の通りである：

1. **理論的貢献**
   - 専門用語を情報理論的圧縮として定式化し、MDL原理に基づく最適化問題として定式化
   - Information Bottleneck Theoryを拡張し、統計的圧縮に論理的制約を統合
   - 確率的型システムによる双方向変換モデルの提案

2. **方法論的貢献**
   - 三層構造の制約検証機構（型チェック層、制約充足層、論理検証層）の設計
   - 動的圧縮理論によるGrokking現象の説明フレームワーク
   - 容量依存圧縮理論によるScaling Lawsとの統合

3. **理論的予測**
   - ハルシネーション削減のための理論的基盤の提供
   - 専門用語処理システムの設計指針の提示
   - 検証可能な予測の明示（第5.5節）

- **第8章（Experimental Evaluation）**: 実証実験の設計と評価計画
